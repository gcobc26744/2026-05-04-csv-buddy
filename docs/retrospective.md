# 開發回顧

## 做得順的地方

- 把功能切成小函式後，反而比較不會卡住：讀檔 → 偵測 delimiter → parse → normalize → summary → write。
- 用一個小的 `sample.csv` 反覆測，很快就能看到清理規則到底有沒有生效。

## 卡住/不確定的地方

- `csv` 的「空白處理」一開始讓我有點困惑：有的情況要靠 `skipinitialspace`，有的情況是自己 `strip()` 比較直覺。
- 欄位名標準化這件事沒有標準答案：做太多會破壞資訊，做太少會讓後續處理很痛苦。

## 我學到什麼

- 標準庫其實就能做不少事，差別在「你願不願意把規則講清楚」。
- 做資料清理時，最好把規則寫得可閱讀、可重現，否則過兩天自己也會忘記為什麼當初這樣做。

## 下一步（如果要繼續）

- 增加基本單元測試（delimiter 偵測、欄位名標準化、數字欄位判斷）
