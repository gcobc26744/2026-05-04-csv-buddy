# 學習筆記：做一個「不用 pandas」的 CSV 小工具

這次刻意只用標準庫，目標不是做出最強的資料處理工具，而是把幾個常遇到的問題拆清楚：

## 1) 分隔符號到底是什麼？

以為 CSV 一定是逗號，但實際上常看到 `;`（尤其是某些地區的匯出檔）。
我在這個小工具裡用一個很土的方法：看前幾行，逗號和分號哪個出現比較多，就假設那個是 delimiter。

我沒有用 `csv.Sniffer` 的原因：
- 它有時候會在「資料量很小」或「內容很不規則」時猜錯
- 我想把判斷邏輯寫得更明確、可讀

## 2) 空值的定義（比想像中麻煩）

最開始我只把 `""` 當成空值，後來才想起來：很多資料會長這樣：

- `" "`（只有空白）
- `"  23"`（前後空白）

所以這次的策略是：
- 每個欄位先 `strip()`
- `strip()` 後如果是空字串，就當作 missing

## 3) 欄位名標準化：要做到什麼程度？

我一開始想要保留原始欄位名，但輸出到新的 CSV 後會變成：
- 讀程式時不方便（`" Name "` 這種 key 很煩）
- 很容易不小心做出兩個看起來一樣、實際上不同的欄位名

所以我選擇把欄位名標準化成：
- 小寫
- 空白改底線
- 除了英數底線以外的字元都移除

這其實會有副作用（例如中文欄位名會被清掉），但對「練習用的工具」我可以接受。
如果要更通用，可能需要更完整的規則或直接保留原名 + 另外建一份 mapping。

## 4) 什麼叫「看起來像數字」？

我的做法：每一欄收集所有非空值，嘗試 `float(value)` 全部成功就把它當 numeric。
這個規則很保守，但好處是簡單、可解釋。

如果想更像真實世界，可能要支援：
- 只要大多數是數字就算 numeric（但要怎麼定義門檻？）
- `1,234` 這種格式
- 貨幣符號
